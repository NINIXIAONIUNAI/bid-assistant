## 一、第一步「招标文件解读」

### 1. 解析中进度反馈
点击"重新解析"或首次解析时，在结果区显示：
```
正在提取项目概况…… ✓
正在识别资格要求…… ✓
正在解析初步评审标准…… ⟳
正在解析评分标准……
正在识别重点响应要求……
正在分析风险提示……
正在提取关键时间节点……
正在整理其他关键信息……
```
逐条点亮（每条 ~400ms），完成后显示 8 个 tab 内容。

### 2. Tab 结构（保持现有 8 个）
项目概况 / 资格要求 / 初步评审标准 / 评分标准 / 重点响应要求 / 风险提示 / 关键时间节点 / 其他关键信息

### 3. 查看来源（新增）
- 每个 tab 顶部右侧新增「📎 查看来源」按钮
- 点击后：左侧上传/粘贴区域 → 切换为 PDF 预览器（用 iframe + mock pdf URL；无 pdf 时显示占位"招标文件.pdf 第 X 页"）
- 跳转到 mock 页码（项目概况→p.1，资格要求→p.5，评分标准→p.12 等）
- 顶部带「← 返回上传区」按钮

## 二、第三步「投标资料配置」（原 Step2Settings 拆分）

把现在的 Step2 拆为：
- **新 Step2 标书基础设置**：标书名称、类型、页数 + 图表与配图（移除清单/引用资料）
- **新 Step3 投标资料配置**（全新）

### 新 Step3 页面结构
顶部蓝色提示条：
> 投标资料将用于生成企业专属内容。未配置资料的章节将保留目录，正文需后续补充。

字段（顺序）：
1. **企业名称*** （输入框）
2. **主营业务** （输入框）
3. **企业简介** [从知识库引用] / [重新上传]
4. **资质证书** [从知识库引用] / [重新上传]
5. **人员配置** [从知识库选择] / [新增人员] —— 选择后显示 chip：张三（总监）×、李四（监理工程师）×
6. **类似业绩** [从知识库选择] / [新增业绩] （多选，chip 展示）
7. **方案模板** [从知识库选择] / [不引用]
8. **项目响应内容*** （根据标书类型动态变化标题）：
   - 服务类 → 上传服务内容文件 / 粘贴服务内容
   - 物资类 → 上传产品参数文件 / 粘贴产品参数
   - 工程类 → 上传工程量清单 / 粘贴工程量内容
   - 其他   → 上传业务资料 / 粘贴业务内容

校验：仅"标书名称"和"项目响应内容"必填。其他未配置时，"下一步"前显示黄色提示卡：
> ⚠ 人员/业绩/方案 未配置。未配置内容对应章节将保留目录，正文需在后续补充。

「从知识库引用/选择」点击 → 打开抽屉，列出 mock 知识库条目，支持单/多选。

原步骤顺序变为：1 招标文件解读 → 2 标书基础设置 → 3 投标资料配置 → 4 编写目录 → 5 编写正文。更新 Stepper。

## 三、知识库模块（全新，侧边栏新增 tab）

### 侧边栏
在「首页 / 我的标书」之间插入「知识库」tab，icon `library_books`，路由 `/bid/library`。

### 路由结构
```
/bid/library             默认页（统计 + 最近使用）
/bid/library/company     企业信息
/bid/library/certs       资质证书
/bid/library/staff       人员信息
/bid/library/cases       类似业绩
/bid/library/templates   方案模板
/bid/library/history     历史标书
```

左侧二级菜单常驻，右侧内容区切换。顶部有搜索框和「+ 新增资料」按钮。

### 各页面（mock 数据 + 表格 + 弹窗交互）
- **默认页**：6 张统计卡片 + 最近使用列表 + "完善知识库可提升AI生成质量"提示
- **企业信息**：表单页（名称/统一社会信用代码/主营业务/简介/LOGO/组织架构图），首次进入显示"暂无企业信息 [立即完善]"
- **资质证书**：表格（证书名称/编号/有效期/状态/操作），上传弹窗带"AI解析中…"过渡，剩余≤30 天黄色提示
- **人员信息**：表格（姓名/岗位/证书数量）+ 详情抽屉
- **类似业绩**：表格（项目名/类型/金额/时间）+ 详情抽屉
- **方案模板**：表格（名称/更新时间）+ 在线预览（mock）
- **历史标书**：表格 + 「从我的标书导入」按钮

所有删除操作 → 二次确认弹窗"删除后不可恢复，确认删除？"
所有保存 → toast "已保存至知识库"
所有空态 → "暂无资料，点击新增"
